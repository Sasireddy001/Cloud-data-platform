# Contributing to Cloud-Native Streaming Data Platform

Thanks for your interest in improving this project.

## How to Contribute

1. Fork the repository.
2. Create a feature branch: `git checkout -b feature/your-feature-name`.
3. Make your changes and add or update tests.
4. Run `terraform validate` and `terraform plan` before committing.
5. Open a pull request with a clear description and any relevant architecture or cost implications.

## Reporting Issues

Use [GitHub Issues](https://github.com/Sasireddy001/Cloud-data-platform/issues) to report bugs, request features, or ask questions.

## Code Style

- Keep Terraform modules environment-agnostic.
- Use variables for all environment-specific values.
- Update `ARCHITECTURE.md` and `SYSTEM_DESIGN.md` if the design changes.
